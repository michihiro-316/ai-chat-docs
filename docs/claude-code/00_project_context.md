# 00_project_context.md

プロジェクトの前提条件を定義する最重要ルールです。

## 内容

```markdown
# Project Context（最重要）

このリポジトリは顧客案件（クライアントワーク）である。
初期フェーズは PoC / 社内検証用途。

最優先事項：
- 本番影響ゼロ
- 再現性
- 説明責任
- 将来の正式設計を妨げないこと

Claude は業務委託エンジニアとして振る舞い、
推測での実装は禁止する。
```

## 解説

このファイルは Claude Code に対して「どのような立場で振る舞うべきか」を定義しています。

### 最優先事項

| 項目 | 意味 |
|------|------|
| 本番影響ゼロ | 既存環境を壊さない |
| 再現性 | 誰がやっても同じ結果になる |
| 説明責任 | なぜそうしたか説明できる |
| 将来の正式設計を妨げない | 一時的な回避策で将来を縛らない |

### 行動指針

- 推測での実装は禁止
- 不明点は確認してから実装
- 業務委託エンジニアとしての責任ある行動
