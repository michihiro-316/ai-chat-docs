# 99_for_non_engineers.md

非エンジニア向けのプロジェクト説明です。

## 目的

エンジニアではない方が以下を理解できることを目的としています：

- このシステムは「何をするものか」
- どのような構成で動いているか
- なぜこの作り方を選んでいるか

## このシステムは何か

**顧客向けに提供する AI チャットアプリ** です。

- ブラウザ（Web画面）から利用
- Googleアカウントでログイン
- 許可されたユーザーのみ利用可能
- AIとチャット形式で会話

## 全体構成

```
画面（HTML）
   ↓
動きを制御する仕組み（JavaScript）
   ↓
判断と処理を行うサーバー（Python）
```

## ファイル構成

```
python/
├─ backend.py      ← サーバー
├─ front.html      ← 画面
├─ script.js       ← 動きの制御
└─ requirements.txt ← 必要な部品一覧
```

## 各ファイルの役割（簡易版）

| ファイル | 役割 | 例え |
|---------|------|------|
| front.html | 見た目 | 建物の外観 |
| script.js | 動き | エレベーターの操作盤 |
| backend.py | 判断 | 管理人室 |

## なぜこう作っているのか

1. **誰が見ても役割がわかる**
2. **間違った変更を防げる**
3. **同じものを何度でも作り直せる**

## 最後に

> 「動けばよい」ではなく「誰が見ても理解でき、再現できる」

これが最優先事項です。
