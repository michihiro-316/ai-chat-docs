# 03_directory_structure.md

ディレクトリ構成を固定で定義します。

## 内容

```markdown
# ディレクトリ構造（固定）

本フェーズでは以下を正式構成とする。

python/
├─ backend.py
├─ front.html
├─ script.js
└─ requirements.txt

肥大化した場合のみ、合意の上で分割する。
```

## 解説

### なぜ固定するのか

| 理由 | 説明 |
|------|------|
| 再現性 | 誰が見ても同じ構成 |
| 説明責任 | 「ここにある」と断言できる |
| シンプルさ | PoCフェーズでは過剰な分割を避ける |

### 各ファイルの役割

| ファイル | 役割 |
|---------|------|
| `backend.py` | サーバー処理（API、認証検証） |
| `front.html` | 画面（DOM構造のみ） |
| `script.js` | クライアントロジック |
| `requirements.txt` | Python依存パッケージ |

### 分割の判断基準

- ファイルが肥大化した場合
- 責務が明確に分離できる場合
- チームで合意が取れた場合

!!! note "ポイント"
    「合意の上で」が重要です。勝手に構成を変えません。
