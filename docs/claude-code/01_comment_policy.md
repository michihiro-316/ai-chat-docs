# 01_comment_policy.md

コードコメントの方針を定義します。

## 内容

```markdown
# コメント方針

コメントは「設計判断のメモ」として扱う。

- なぜ必要かを書く
- 判断を伴う箇所に限定
- 書きすぎない

自明な処理への冗長なコメントは禁止。
```

## 解説

### 良いコメント

```python
# Firebase Admin SDKではなくクライアント検証を使用
# 理由：PoCフェーズでは運用負荷を下げるため
def verify_token(token):
    ...
```

### 悪いコメント

```python
# ユーザーを取得する
user = get_user(id)  # 自明なので不要

# iを1増やす
i += 1  # 自明すぎる
```

### 方針のポイント

| DO | DON'T |
|----|-------|
| なぜその判断をしたか書く | 何をしているか書く（コードを読めばわかる） |
| 例外的な処理に注釈を付ける | すべての関数にコメントを付ける |
| 将来の注意点を残す | 自明なことを説明する |
