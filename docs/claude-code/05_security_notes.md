# 05_security_notes.md

PoC / 顧客提供向けのセキュリティ方針を定義します。

## 内容

```markdown
# セキュリティ方針（PoC / 顧客提供向け）

## 基本スタンス

本プロジェクトは顧客に提供されるため、
最低限のセキュリティは必須とする。

ただし、以下を同時に満たすことを重視する：

- 運用者本人が理解・説明できる
- 過度に複雑化しない
- 将来の強化を妨げない
```

## このフェーズで必ず守ること

### 認証・認可

| 項目 | 実装 |
|------|------|
| 認証基盤 | Firebase Authentication |
| JWT取得 | フロントエンドで実施 |
| JWT検証 | バックエンドで実施 |
| アクセス制御 | メール/ドメイン制御 |

### 秘密情報の管理

| DO | DON'T |
|----|-------|
| 環境変数で管理 | コードに直書き |
| サーバー側で保持 | フロントに渡す |
| ログから除外 | ログに出力 |

## このフェーズでやらないこと

- JWT署名の完全検証（Admin SDK）
- WAF / IAM の高度設定
- IP制限・多要素認証
- 監査ログの完全実装

## 判断基準

以下すべてに YES と答えられること：

- [x] 自分で説明できるか？
- [x] 障害時に自分で直せるか？
- [x] 顧客に「今はここまで」と説明できるか？
