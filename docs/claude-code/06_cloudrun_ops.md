Cloud Runの運用方針を定義します。

## 内容

```markdown
# Cloud Run 運用方針（シンプル重視）

## 基本方針

Cloud Run は以下を目的として利用する。

- 管理負荷を最小化する
- 運用を単純に保つ
- 本人が把握できる範囲に留める
```

## 必須ルール

| ルール | 説明 |
|--------|------|
| 単一サービス構成 | 分割しない |
| フロント・バック同居 | 同一Cloud Runに配置 |
| 環境変数で設定管理 | Secret Managerは使わない |
| PORT環境変数を使用 | Cloud Runが自動設定 |

## ログ・監視

| DO | DON'T |
|----|-------|
| print/loggingは最小限 | 過剰なログ出力 |
| Cloud Loggingで確認 | 独自の監視システム構築 |
| エラー時のみ詳細ログ | すべてのリクエストをログ |

## このフェーズでやらないこと

- 複数サービス分割
- 複雑な IAM 設計
- 高度なオートスケーリング調整
- マルチリージョン構成

## 判断基準

- [x] 本人がCloud Runの状態を説明できるか
- [x] 障害時に一人で切り分けできるか

YES であれば採用する。
